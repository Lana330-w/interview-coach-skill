# Question Bank by Role Type

Read this file when you need inspiration for question patterns. Always customize questions to the candidate's actual resume — never use these verbatim.

---

## Software Engineering / 软件工程

### Technical Questions
- 描述你在 [项目名] 中最复杂的技术挑战，你是如何解决的？
- 你如何在 [技术A] 和 [技术B] 之间做技术选型？用你简历中的某个项目举例
- 解释一下你在 [X] 项目中的系统架构，如果重新设计你会改变什么？
- 你用过哪些性能优化手段？在你哪个项目里用到了？
- 说说你对 [候选人简历中出现的技术] 最深的理解

### System Design
- 如果让你设计 [与岗位相关的系统]，你会怎么考虑？
- 你的 [某项目] 如何处理高并发/大数据量的场景？
- 说说你们团队的部署流程是什么样的

### Code Quality
- 你如何保证代码质量？有没有在你们团队推行过什么实践？
- 聊聊你做过的最难 debug 的一次经历

## Product Manager / 产品经理

### Product Thinking
- 讲一个你主导过的产品需求，从发现问题到上线的全过程
- 你如何衡量一个功能是否成功？
- 如果用户数据和用户反馈相矛盾，你怎么处理？
- 描述你如何平衡技术债务和新功能需求

### Stakeholder Management
- 你如何与强势的研发/设计团队协作？有没有发生过分歧？
- 你如何向管理层汇报产品进展？

### Analytical
- 你的 [某产品功能] 上线后核心指标是什么？结果怎么样？
- 如何判断一个功能值不值得做？

## Data / 数据分析

### Technical
- 你在 [项目] 中用了哪些数据工具和语言？
- 说说你做过最复杂的一次数据分析，方法论是什么？
- 如何处理数据缺失和异常值？举个你的实际例子

### Business Acumen
- 你的分析结果如何影响了业务决策？
- 你如何向非技术背景的同事展示数据发现？

---

## Design / 设计

### Portfolio-based
- 讲一个你最骄傲的设计项目，从用户研究到最终方案
- 你的设计方案被推翻过吗？怎么处理的？

### Process
- 你的设计流程是什么？
- 如何在设计美感和商业目标之间取得平衡？

## Behavioral Questions (通用行为题)

### Leadership
- 描述一次你带领团队克服困难的经历
- 你如何处理团队成员绩效不达标的情况？

### Conflict
- 你和同事/老板产生过严重分歧吗？怎么解决的？
- 有没有坚持过一个最终被证明是正确的不受欢迎的观点？

### Growth
- 你过去一年学到的最重要的东西是什么？
- 你的职业规划是什么？为什么选择这个岗位？

### Failure
- 描述一次你的失败，你学到了什么？
- 你做过最后悔的职业决定是什么？

## Seniority Level Calibration

### Junior (0–2 years)
- Focus on: fundamentals, learning speed, attitude, basic project experience
- Avoid deep system design questions
- Probe: how they learn, how they handle mistakes

### Mid-level (3–5 years)
- Focus on: ownership, technical depth, cross-team work
- Include: some system design, technical tradeoffs
- Probe: specific technical decisions they made

### Senior (5+ years)
- Focus on: impact, leadership, architecture, business alignment
- Include: org-level thinking, mentorship, technical vision
- Probe: how they influenced others, built for scale, managed complexity

### Manager/Lead
- Focus on: people management, strategy, cross-functional alignment
- Include: hiring, performance management, roadmap ownership
- De-emphasize: individual technical implementation details
